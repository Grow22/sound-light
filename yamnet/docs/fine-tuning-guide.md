# YAMNet 파인튜닝 가이드

## 1. 파인튜닝이란?

YAMNet은 원래 521가지 소리를 분류하는 모델입니다.
하지만 우리 프로젝트는 **6가지 소리만 정확하게** 분류하면 됩니다.

파인튜닝은 YAMNet의 **소리를 분석하는 능력은 그대로 유지**하면서,
마지막 분류 부분만 우리 목적에 맞게 **새로 학습**시키는 방법입니다.

> 비유: YAMNet = 소리를 잘 듣는 "귀", 파인튜닝 = 우리 소리를 판별하는 "뇌"를 새로 붙이는 것

---

## 2. 우리가 분류하는 6가지 소리

| 카테고리 | 원본 데이터 수 | 설명 |
|---------|-------------|------|
| 개 짖음 | 10개 | 허스키, 시바견 등 |
| 노크 소리 | 7개 | 문 두드리는 소리 |
| 물소리 | 7개 | 수도꼭지, 물 흐르는 소리 |
| 사이렌 | 7개 | 경보, 사이렌 소리 |
| 아기 울음 | 10개 | 영아 울음소리 |
| 초인종 | 5개 | 초인종, 벨소리 |
| **합계** | **46개** | |

---

## 3. 데이터 증강 (Data Augmentation)

### 왜 증강이 필요한가?

원본 데이터가 **46개로 매우 적습니다.**
데이터가 적으면 모델이 학습 데이터만 외워버리고(과적합),
실제 환경에서는 제대로 인식하지 못합니다.

이를 해결하기 위해 원본 소리를 다양하게 변형해서 학습 데이터를 늘립니다.
**원본 1개 → 10개로 증강하여, 총 46개 → 460개로 확장합니다.**

### 10가지 증강 방법

하나의 원본 소리에서 아래 10가지 버전을 생성합니다:

| 번호 | 증강 방법 | 설명 | 왜 필요한가? |
|-----|----------|------|------------|
| 1 | **원본** | 변형 없이 그대로 사용 | 기본 학습 데이터 |
| 2 | **약한 노이즈** | 작은 잡음 추가 | 조용한 환경의 배경 소음 대비 |
| 3 | **강한 노이즈** | 큰 잡음 추가 | 시끄러운 환경에서도 인식하도록 |
| 4 | **볼륨 작게** | 음량 60%로 축소 | 먼 거리의 소리 대비 |
| 5 | **볼륨 크게** | 음량 140%로 증가 | 가까운 거리의 큰 소리 대비 |
| 6 | **시간 이동(앞)** | 소리를 0.2초 앞으로 밀기 | 소리 시작 시점이 달라도 인식하도록 |
| 7 | **시간 이동(뒤)** | 소리를 0.2초 뒤로 밀기 | 소리 시작 시점이 달라도 인식하도록 |
| 8 | **빠르게 재생** | 속도 약간 빠르게 (피치 높아짐) | 높은 톤의 소리 변형 대비 |
| 9 | **느리게 재생** | 속도 약간 느리게 (피치 낮아짐) | 낮은 톤의 소리 변형 대비 |
| 10 | **구간 마스킹** | 0.1초 구간을 무음 처리 | 소리 일부가 끊겨도 인식하도록 |

### 증강 결과

```
원본 46개 × 10가지 변형 = 총 460개 학습 데이터

카테고리별:
  개 짖음:   10개 → 100개
  노크 소리:  7개 →  70개
  물소리:     7개 →  70개
  사이렌:     7개 →  70개
  아기 울음: 10개 → 100개
  초인종:     5개 →  50개
```

## 4. 파인튜닝 과정

```
[원본 소리 46개]
    ↓ 데이터 증강 (10배)
[증강된 소리 460개]
    ↓ YAMNet으로 특징 추출 (1024차원 벡터)
[임베딩 데이터 460개]
    ↓ 학습/검증 분리 (80% / 20%)
[학습: 368개 / 검증: 92개]
    ↓ 분류 모델 학습 (Dense 레이어)
[파인튜닝 완료]
    ↓
[모델 저장 (.keras, .tflite)]
```

## 5. Threshold (임계값)

### 왜 필요한가?

우리 모델은 6가지 소리만 학습했기 때문에, 어떤 소리가 들어와도 **6개 중 하나로 분류**합니다.
예를 들어 TV 소리가 들어와도 억지로 "초인종 45%"처럼 판정합니다.

이런 **오알림을 방지**하기 위해 threshold(임계값)를 설정합니다.

### 동작 방식

```
Threshold = 90%

소리 입력 → 모델 예측 → 확률 90% 이상?
  → Yes: 알림 전송 (예: "초인종이 울렸습니다!")
  → No:  무시 (알림 없음)
```

### 예시

| 입력 소리 | 모델 판정 | 확률 | 결과 |
|----------|----------|------|------|
| 진짜 초인종 | 초인종 | 99% | 90% 이상 → **알림** |
| 진짜 사이렌 | 사이렌 | 98% | 90% 이상 → **알림** |
| TV 벨소리 | 초인종 | 45% | 90% 미만 → **무시** |
| 자동차 경적 | 사이렌 | 60% | 90% 미만 → **무시** |

> 우리 기기의 목적은 **모든 소리를 분류하는 것이 아니라**,
> 청각장애인이 놓치면 안 되는 중요한 소리만 정확하게 감지하여 알려주는 것입니다.
> 따라서 모든 소리를 수집할 필요 없이, threshold로 확신하는 소리에만 반응하면 됩니다.

- Threshold 90% 적용 시 모든 카테고리 정상 감지 (97~100%)
