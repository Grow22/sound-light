# YAMNet 소리 분류 모델 개요

## YAMNet이란?

Google이 만든 **소리 분류 AI 모델**입니다.
대규모 오디오 데이터(AudioSet)로 학습되어 있으며, **521종류의 소리**를 구분할 수 있습니다.

예시: Speech, Music, Dog bark, Fire alarm, Doorbell, Water, Siren 등

---

## 우리 프로젝트에서의 활용

### 목표
YAMNet을 활용하여 아래 **5가지 소리**를 정확하게 분류하는 것

| 카테고리 | YAMNet 521개 중 관련 클래스 |
|----------|---------------------------|
| 사이렌/화재벨 | fire alarm, alarm, siren, smoke detector |
| 개 소리 | dog, bark, howl, growl |
| 아기 울음 | baby cry, infant cry |
| 초인종 | doorbell, ding-dong, bell |
| 노크/현관문 | door, knock, slam |

### 현재 문제
YAMNet은 521개 소리를 **범용적으로** 구분하다 보니, 우리 상황에 딱 맞지 않는 경우가 있습니다.

예) 사이렌 소리를 넣었는데 "Music"으로 인식되는 경우 발생

## 파인튜닝 (Fine-tuning)

### 파인튜닝이란?
이미 학습된 모델을 가져와서 **우리 목적에 맞게 추가 학습**시키는 것입니다.

| 방식 | 설명 | 필요 데이터 |
|------|------|------------|
| 처음부터 학습 | 빈 모델에 전부 학습 | 수만~수백만 개 |
| **파인튜닝** | 학습된 모델 위에 추가 학습 | **수십~수백 개** |

### 파인튜닝 과정

```
[오디오 파일] → [YAMNet 임베딩 추출] → [우리 분류기 학습] → [5가지 소리 분류 모델]
                 (고정, 건드리지 않음)    (이 부분만 새로 학습)
```

1. **데이터 수집** - 각 카테고리별 오디오 파일 준비 (카테고리당 최소 20~50개 권장)
2. **전처리** - 오디오를 16kHz로 변환, 일정 길이로 자르기
3. **임베딩 추출** - YAMNet에 오디오를 통과시켜 특징 벡터(숫자 배열) 추출
4. **분류기 학습** - 추출된 특징 벡터로 우리 분류기(Dense 레이어) 학습
5. **평가** - 정확도 확인, 혼동 행렬로 어떤 소리가 잘못 분류되는지 파악

### 쉽게 비유하면

> YAMNet = "소리를 듣고 특징을 파악하는 귀"
>
> 파인튜닝 = "그 귀가 들은 특징을 우리가 원하는 5가지로 판단하는 뇌를 붙이는 것"
>
> 귀(YAMNet)는 이미 잘 훈련되어 있으니, 뇌(분류기)만 새로 학습시키면 됩니다.

## 전체 프로젝트 흐름

```
1. 데이터 수집 (각 조원이 카테고리별 소리 파일 준비)
        ↓
2. Google Colab에서 파인튜닝 (GPU 활용)
        ↓
3. 학습된 모델 파일 (.h5) 저장
        ↓
4. 라즈베리파이에 모델 배포 → 실시간 소리 감지
```

- 학습(파인튜닝)은 **Google Colab**에서 진행 (GPU 필요)
- 학습된 모델로 소리 판별하는 것은 **라즈베리파이**에서도 가능 (추론만 하면 되므로)
- 데이터는 한번 모아두면 나중에 모델 개선할 때 계속 재사용 가능
