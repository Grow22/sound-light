# 기술 세부사항 (논문 작성용)

## 1. 시스템 전체 구조

### 2단계 분류 파이프라인
```
오디오 입력 (INMP441 I2S 마이크, 48kHz)
    ↓ 리샘플링 (48kHz → 16kHz)
    ↓
━━━ 1단계: YAMNet (521개 카테고리) ━━━
    ├─ [무시] 대화, 음악, 웃음, 동물(개 제외), 날씨, 차량 → 차단
    ├─ [직접 매핑] 개짖음, 물소리, 인터폰, 노크, 창문깨짐
    │   → 신뢰도 ≥ 0.2 → 서버 전송
    └─ [2차 분류기 전달] 도어, 비프, 사이렌, 전화 관련 소리
        ↓
━━━ 2단계: Hearo 분류기 (11개 카테고리) ━━━
    → YAMNet 1024차원 임베딩 추출
    → 11개 카테고리 분류
    → 신뢰도 ≥ 0.2 → 서버 전송
```

### 2단계 구조가 필요한 이유
- **문제**: 단일 분류기(11개)는 모든 소리를 11개 중 하나로 강제 분류
  - 대화 → "사이렌" (60%), 음악 → "벨소리" (70%) 같은 오분류 발생
- **해결**: YAMNet이 521개 카테고리로 1차 필터링하여 관련 없는 소리 차단
  - 관련 소리만 2차 분류기로 전달 → 오탐률 대폭 감소

|---------------|-------------------|------|
| 도어락 | 6개 (Door, Slam 등) | 한국 도어락 특유 소리 |

- 초인종: 인터폰과 통합 (YAMNet이 둘 다 Telephone로 분류)

## 3. 오디오 녹음 파라미터

| 파라미터 | 값 | 설명 |
|---------|-----|------|
| 마이크 | INMP441 (I2S MEMS) | 저가/고감도 |
| 마이크 샘플링 | 48,000 Hz | INMP441 기본 |
| YAMNet 입력 | 16,000 Hz | 모델 요구사항 |
| 리샘플링 | scipy.signal.resample | 48k→16k 선형 |
| 녹음 시간 | 1.0초 | 추론 단위 |
| 오디오 형식 | float32 [-1.0, 1.0] | 정규화 |
| 채널 | 1 (모노) | - |
| 무음 기준 | RMS < 0.003 | 무음이면 건너뜀 |

## 4. 모델 상세

### YAMNet (1단계)
- **파일**: yamnet.tflite (16 MB)
- **학습 데이터**: Google AudioSet (YouTube 200만개+ 오디오)
- **출력 1**: 521개 카테고리 확률 (scores)
- **출력 2**: 1024차원 임베딩 벡터 (음향 특성)
- **아키텍처**: MobileNet 기반
- **역할**: 필터링 + 임베딩 추출

### Hearo 분류기 (2단계)
- **파일**: hearo_classifier.tflite (298 KB)
- **입력**: YAMNet의 1024차원 임베딩
- **출력**: 11개 카테고리 확률 (softmax)
- **구조**: Dense(256) → Dropout → Dense(128) → Dropout → Dense(11, softmax)
- **학습 방식**: YAMNet 임베딩은 고정, Dense 레이어만 학습
- **학습 데이터**: 한국 가정 환경음 직접 수집
  - 원본: 311개 (팀원 직접 녹음)
  - 증강 후: 2,480개 (Train), 63개 (Val, 원본만)
- **검증 정확도**: 90.5% (11개 카테고리)
- **신뢰도 임계값**: 0.2 (YAMNet 점수 특성 반영)

### 데이터 증강 기법 (10가지)
1. 가우시안 노이즈 추가
2. 볼륨 변경
3. 시간 이동 (Time Shift)
4. 속도 변경
5. 피치 변경
6. 시간 마스킹
7. 주파수 마스킹
8. 시간 스트레칭
9. 반향 추가
10. 배경 노이즈 합성

### Data Leakage 방지
- **문제**: 증강 후 Train/Val 분리 시 같은 원본의 변형이 양쪽에 포함
- **해결**: 원본 단위로 먼저 80:20 분리 → Train에만 증강 적용
- **Val 구성**: 63개 원본 소리만 (증강 없음)

## 6. 하드웨어 구성

### Hub-Node 구조
- **Hub**: Raspberry Pi 4 (AI 추론 담당)
- **Node**: ESP32 + INMP441 마이크 (각 방에 설치, 추후)
- **현재**: RPi에 직접 마이크 연결 (단일 공간 테스트)

### 비용 비교
| 구성 | 비용 | 비고 |
|------|------|------|
| 기존 (RPi 3대) | ~465,000원 | 방마다 RPi |
| Hub-Node (RPi 1대 + ESP32 3대) | ~254,000원 | 45% 절감 |
| 방 추가 시 | +30,000원 | ESP32 1대 추가 |

### 알림 방식
- **LED**: WS2812B 스트립 (시각 알림)
- **진동**: 진동 모터 (촉각 알림, 수면 중 활용)
- **푸시 알림**: Firebase FCM → 스마트폰 (외출 시)

## 10. 핵심 수치 (논문에 인용할 것)

- Hub-Node 구조: 기존 대비 45% 비용 절감
- 검증 정확도: 90.5% (11개 카테고리)
- 추론 사이클: ~1.5-2초 (실시간 가능)
