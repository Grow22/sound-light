## 1. 전체 구조

```
소리 입력 → YAMNet (음향 패턴 추출) → 분류기 (가중치 적용) → 카테고리 판단
```

- **YAMNet**: 소리를 1024개의 숫자(음향 패턴)로 변환하는 역할. 이미 학습된 모델이라 우리가 건드리지 않음.
- **분류기 (Dense 레이어)**: 음향 패턴에 가중치를 곱해서 11개 카테고리 중 어떤 소리인지 판단. 이 가중치를 학습시키는 게 파인튜닝.

---

## 2. Train / Val 데이터 역할

| | Train (학습 데이터) | Val (검증 데이터) |
|---|---|---|
| 역할 | 모델의 가중치를 학습시킴 | 학습된 모델이 잘 작동하는지 시험 |
| 데이터 수 | 2,480개 (증강 적용) | 63개 (원본만 사용) |
| 모델이 본 적 있는가 | 있음 | **없음** |
| 비유 | 수학 문제집으로 공부 | 모의고사 |

### 왜 8:2로 나누는가

- 학습 데이터를 많이 주면 → 모델이 잘 배우지만 검증 데이터 부족
- 검증 데이터를 많이 주면 → 평가는 정확하지만 모델이 못 배움
- 8:2가 이 균형점으로 가장 보편적인 비율

### 왜 Val에 증강을 안 하는가

증강 데이터는 원본의 변형이라 모델이 비슷한 패턴을 이미 봤을 수 있음.
원본 그대로 검증해야 실제 환경에서의 성능에 가까운 결과가 나옴.

### Data Leakage 방지

원본 단위로 Train/Val을 먼저 분리한 후 Train에만 증강을 적용.
증강 후에 분리하면 같은 원본의 변형이 Train과 Val에 모두 들어가서 검증 결과를 신뢰할 수 없음.

### Val은 실전에서 쓰이지 않는다

```
[개발 단계] (한 번만 수행)
Train 2,480개로 학습 → Val 63개로 검증 → "90.5%면 OK" → 모델 저장

[실전 단계] (계속 반복)
마이크에 소리 들어옴 → 저장된 모델이 바로 분류 → "초인종 97%" → 알림
```

Val과 비교하는 과정은 실전에 없음. 모델은 이미 학습 완료된 상태.

## 3. 정답은 어떻게 결정되는가

폴더 구조가 곧 정답지이다.

```

├── 노크\         ← 이 폴더의 파일은 전부 정답 = "노크"
├── 초인종\       ← 이 폴더의 파일은 전부 정답 = "초인종"

└── ...
```

코드에서 폴더 이름을 카테고리로 자동 매핑함.

## 4. 음향 패턴이란?

소리를 YAMNet에 넣으면 1024개의 숫자가 나옴. 이 숫자들이 소리의 특징을 나타냄.

```
노크 소리   → YAMNet → [0.2, 0.8, 0.1, 0.5, ...]  (1024개)
초인종 소리 → YAMNet → [0.7, 0.3, 0.9, 0.2, ...]  (1024개)
```

- 이 값은 소리가 같으면 항상 같음 (학습과 무관)
- 각 위치는 소리의 수학적 특징 (반복성, 음높이, 세기 등)
- YAMNet이 자동으로 추출하며, 사람이 의미를 정하는 게 아님

## 5. 가중치란?

"이 음향 패턴이 들어왔을 때 어떤 카테고리로 판단할 것인가"의 기준 숫자.

**11개 카테고리마다 가중치 세트가 따로 존재한다:**

```
노크 가중치:   [0.1, 0.9, -0.3, 0.2, ...]  (1024개)
초인종 가중치: [-0.2, 0.3, 0.8, -0.1, ...]  (1024개)
사이렌 가중치: [0.5, -0.1, 0.2, 0.7, ...]   (1024개)
... (11개 카테고리 × 1024개 = 총 11,264개 가중치)
```

각 원소의 의미: **"이 위치의 음향 특징이 해당 카테고리 판단에 얼마나 중요한가"**

```
노크 가중치: [0.1, 0.9, -0.3, 0.2]
                   ↑
              "반복성이 높으면 노크일 가능성 높다"
              → 0.9 (높은 가중치)

사이렌 가중치: [0.5, -0.1, 0.2, 0.7]
                ↑              ↑
           "음높이 중요"   "세기 중요"
```

## 6. 점수 계산 과정 (소리 가중치 예시)

### 노크 소리가 들어왔을 때

```
노크 음향 패턴: [0.2, 0.8, 0.1, 0.5]

노크 가중치:   [0.1, 0.9, -0.3, 0.2]
노크 점수 = 0.2×0.1 + 0.8×0.9 + 0.1×(-0.3) + 0.5×0.2
         = 0.02 + 0.72 + (-0.03) + 0.10
         = 0.81  ← 높음!

초인종 가중치: [-0.2, 0.3, 0.8, -0.1]
초인종 점수 = 0.2×(-0.2) + 0.8×0.3 + 0.1×0.8 + 0.5×(-0.1)
           = (-0.04) + 0.24 + 0.08 + (-0.05)
           = 0.23  ← 낮음

→ 노크 점수(0.81) > 초인종 점수(0.23) → "노크"로 판단
```

### 같은 가중치에 초인종 소리가 들어왔을 때

```
초인종 음향 패턴: [0.7, 0.3, 0.9, 0.2]

노크 점수 = 0.7×0.1 + 0.3×0.9 + 0.9×(-0.3) + 0.2×0.2
         = 0.07 + 0.27 + (-0.27) + 0.04
         = 0.11  ← 낮음

초인종 점수 = 0.7×(-0.2) + 0.3×0.3 + 0.9×0.8 + 0.2×(-0.1)
           = (-0.14) + 0.09 + 0.72 + (-0.02)
           = 0.65  ← 높음!

→ 초인종 점수(0.65) > 노크 점수(0.11) → "초인종"으로 판단
```

**같은 가중치인데 음향 패턴이 다르니까 결과가 달라진다.**
이것이 데이터 수에 관계없이 소리를 구분할 수 있는 원리.

## 7. 학습(가중치 조정) 과정 예시

### 맨 처음 (가중치 랜덤)

```
노크 가중치 (랜덤):  [0.01, 0.01, 0.01, 0.01]
노크 소리 음향 패턴:  [0.2,  0.8,  0.1,  0.5]

노크 점수 = 0.2×0.01 + 0.8×0.01 + 0.1×0.01 + 0.5×0.01 = 0.016
→ 점수 낮음, 다른 카테고리가 더 높을 수 있음 → 틀림
```

### 가중치 조정 원리

```
정답: 노크 100%
예측: 노크 점수가 낮음 → 올려야 함

"어떤 위치의 가중치를 올리면 점수가 가장 많이 오르는가?"

음향 패턴: [0.2, 0.8, 0.1, 0.5]
                  ↑
             0.8이 가장 큼
→ 이 위치의 가중치를 올리면 점수가 가장 많이 상승
→ 0.8 × (가중치 0.01 올림) = 점수 0.008 상승
→ 0.2 × (가중치 0.01 올림) = 점수 0.002 상승

결과: 2번째 위치의 가중치가 가장 많이 올라감
```

### 조정 결과

```
노크 가중치: [0.01, 0.01, 0.01, 0.01]
                    ↓
노크 가중치: [0.02, 0.08, 0.02, 0.04]
                    ↑↑↑
              반복성(0.8) 위치 가중치가 가장 많이 올라감
```

### 이걸 2,480개 × 28바퀴 반복하면

```
1바퀴째: 거의 찍는 수준 → 정확도 68%
5바퀴째: 패턴이 보이기 시작 → 정확도 93%
28바퀴째: 대부분 맞춤 → 정확도 ~100% (Train 기준)
```

### 가중치 조정의 핵심: 바뀌는 건 가중치이지 소리 패턴이 아니다

```
노크 소리 넣음 → 예측: 초인종 40%, 노크 16%

바뀌는 것 (가중치):
  초인종 가중치: [0.02, 0.02, 0.02, 0.02] → 줄임
  노크 가중치:   [0.01, 0.01, 0.01, 0.01] → 높임

안 바뀌는 것 (소리 패턴):
  음향 패턴: [0.2, 0.8, 0.1, 0.5] → 항상 고정
```

### 맞아도 조정한다

가중치 조정은 틀렸을 때만 하는 게 아니라 **항상** 한다. 조정 크기가 다를 뿐이다.

```
노크 소리 넣음 → "노크 99%" → 거의 정답 → 가중치 아주 살짝 조정
노크 소리 넣음 → "노크 60%" → 맞긴 함   → 가중치 중간 정도 조정
노크 소리 넣음 → "초인종 70%" → 틀림    → 가중치 크게 조정
```

| 상황 | 목표와의 차이 | 조정 크기 |
|------|------------|---------|
| 완전히 틀림 (정답 카테고리 16%) | 84% 부족 | 크게 |
| 맞긴 한데 애매 (정답 카테고리 60%) | 40% 부족 | 중간 |
| 거의 완벽 (정답 카테고리 99%) | 1% 부족 | 아주 살짝 |

목표(정답 카테고리 100%)와 예측의 차이가 클수록 크게 조정하고, 차이가 작을수록 살짝만 조정한다.

### 조정 방향 정리

소리를 넣었을 때 정답이 아닌 카테고리의 확률이 더 높으면:

1. **정답 카테고리의 가중치** → 높임 (해당 소리 패턴에 더 잘 반응하도록)
2. **오답 카테고리의 가중치** → 줄임 (해당 소리 패턴에 덜 반응하도록)

이때 소리 패턴 값이 큰 위치의 가중치가 더 많이 조정된다 (곱셈의 수학적 성질).

```
예시: 노크 소리 [0.2, 0.8, 0.1, 0.5] 넣었는데 초인종으로 오분류

정답(노크) 가중치 조정:
  1번째(×0.2): 조금 올림
  2번째(×0.8): 많이 올림  ← 음향 패턴 값이 크니까
  3번째(×0.1): 아주 조금 올림
  4번째(×0.5): 중간 올림

오답(초인종) 가중치 조정:
  1번째(×0.2): 조금 내림
  2번째(×0.8): 많이 내림  ← 음향 패턴 값이 크니까
  3번째(×0.1): 아주 조금 내림
  4번째(×0.5): 중간 내림
```

이 과정을 수만 번 반복하면 각 카테고리의 가중치가 해당 소리의 고유한 음향 특징에 최적화된다.

## 8. 데이터 수 불균형의 영향

### 데이터가 많으면 그 카테고리 쪽으로 편향되는가?

**소리 패턴이 확연히 다르면: 아니다**
**소리 패턴이 비슷하면: 그렇다**

| 상황 | 데이터 수 영향 | 실제 결과 |
|------|-------------|----------|
| 물소리 vs 노크 (완전히 다른 패턴) | 거의 없음 | 둘 다 100% |
| 초인종 vs 인터폰 (비슷한 벨소리) | 데이터 많은 쪽으로 쏠림 | 초인종 57% |

### 왜 이런가?

가중치는 음향 패턴의 각 위치에 대해 조정되므로, 패턴이 다르면 서로 다른 위치의 가중치에 영향을 줌.
하지만 비슷한 소리는 비슷한 위치에 높은 값이 있어서, 데이터 많은 쪽의 가중치가 더 강하게 조정됨.

### 그래도 최소한의 데이터는 필요하다

패턴이 단순한 소리라도 가중치가 충분히 조정되려면 어느 정도의 학습 데이터는 필요함.

## 9. Threshold (신뢰도 임계값) 90%

학습되지 않은 소리가 들어왔을 때를 대비.

```
TV 소리 입력 → 모델: "초인종 45%, 사이렌 30%, ..." → 90% 미만 → 알림 안 함
실제 초인종  → 모델: "초인종 97%" → 90% 이상 → 알림 전송
```

## 10. 발표 시 예상 질의응답

**Q: 왜 YAMNet을 선택했나요?**
A: 경량 모델이라 Raspberry Pi에서 실시간 추론 가능. AudioSet 기반 사전학습으로 소리 특징 추출 능력이 검증됨.

**Q: Data Leakage가 뭔가요? 어떻게 해결했나요?**
A: 증강 후에 Train/Val을 나누면 같은 원본의 변형이 양쪽에 들어가 검증 결과를 신뢰할 수 없는 문제. 원본 단위로 먼저 분리한 후 Train에만 증강을 적용하여 해결.

**Q: 초인종과 인터폰이 왜 헷갈리나요?**
A: 둘 다 벨소리 계열이라 음향 특성이 유사. 데이터를 더 수집하거나 소리 전처리 방법을 개선하면 해결 가능.

**Q: 왜 K-Fold Cross Validation을 안 썼나요?**
A: 원본 단위로 Data Leakage를 방지해야 했기 때문에 Hold-out 방식을 채택. 데이터 확보 후 K-Fold 적용 계획.

**Q: 클래스 불균형 문제는 어떻게 대응하나요?**
A: 현재 인지하고 있으며, 향후 데이터 수 균등화 또는 Class Weight 적용을 계획.

**Q: 검증 정확도 90.5%를 어떻게 개선할 계획인가요?**
A: 감지 대상 15종 확대와 함께 카테고리당 데이터 수를 늘리고, 비슷한 소리 간 구분력을 높이기 위한 전처리 기법 적용 예정.
